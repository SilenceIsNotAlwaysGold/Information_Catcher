## 1. TrendPulse 是什么

TrendPulse 是一个**多平台内容监控与发布**的 SaaS 平台，目前正式支持小红书、抖音两个平台。

| 你能做的事 | 适用人群 |
|------|------|
| 盯自家账号的数据涨跌（点赞/收藏/评论），有动静推送到飞书 / 企业微信 | 自媒体运营、内容团队 |
| 订阅竞品博主的新作品，第一时间收到通知 | 选题策划、对标分析 |
| 按关键词抓取全网爆款，按点赞数过滤 | 选题灵感、热点追踪 |
| 看自己作品的最新评论，AI 自动改写文案 | 内容创作者 |
| 一键发布到小红书 / 抖音（开发中） | 跨平台分发 |

**核心理念**：所有抓取都通过你自己浏览器里登录的扩展进行，平台风控规则下风险极低。

## 2. 注册与登录

1. 打开 TrendPulse 网址（由我们提供给你）
2. 点「注册」，用邀请码注册（如果有），填用户名、邮箱、密码
3. 登录后进入 Dashboard
4. 首次登录建议进「个人设置」修改密码 + 完善信息

---

## 3. 安装 TrendPulse Helper 浏览器扩展（必装）

**为什么必装**：所有抓取（搜索、追新、评论、详情、发布）都是通过这个扩展在你已登录的浏览器里执行的，绕开 cookie 共享带来的封号风险。

### 安装步骤

1. Dashboard 左侧菜单 → **「我的浏览器扩展」**
2. 点「下载扩展包」，得到 `pulse-helper.zip`
3. 解压到任意目录（建议放固定位置，例如 `~/Documents/pulse-helper/`）
4. Chrome 打开 `chrome://extensions/`，右上角打开「开发者模式」
5. 点「加载已解压的扩展程序」，选刚解压的目录
6. 扩展列表里能看到 **TrendPulse Helper**，点固定到工具栏

### 绑定服务器

1. 点工具栏的 TrendPulse Helper 图标
2. 「服务器地址」填你访问 Dashboard 的网址（例如 `https://trendpulse.example.com`）
3. 「Token」从 Dashboard 「我的浏览器扩展」页面复制
4. 点「保存并连接」，看到绿色「已连接」就 OK

### 登录目标平台

在同一个浏览器里登录你要监控的平台账号：
- 小红书：https://www.xiaohongshu.com/
- 抖音：https://www.douyin.com/

> 扩展不会读你的 cookie 上传到服务器，它只是在你浏览的页面上抓 fetch 响应，所以**必须保持浏览器打开 + 已登录**才能正常抓数据。

## 4. 三大核心板块（每个平台都有）

左侧菜单的「小红书 / 抖音」两个板块结构完全一致，每个都包含：

### 4.1 监控帖子

把你（或他人）的帖子 URL 录进来，平台会按设定频率（默认 30 分钟）抓取互动数据，超阈值发告警。

**怎么用**：
1. 进入「小红书 → 监控帖子」
2. 顶部「订阅博主」… 等等，监控帖子是另一个动作 → 点「+ 添加帖子」
3. 粘贴一个或多个帖子链接（每行一个），可选择「分组」
4. 保存后，调度器会在下个周期开始抓取
5. 每条帖子都会显示当前点赞 / 收藏 / 评论数 + 失效状态

**告警**：当点赞 / 收藏 / 评论增量超过阈值，会推送到飞书或企微（见 §5）。

### 4.2 热门内容

按关键词在该平台搜索热门作品，按点赞数过滤后入库。

**怎么用**：
1. 进入「小红书 → 热门内容」
2. 点右上角「追新设置」（齿轮图标）
3. 配置关键词列表（每行一个）+ 最低点赞数 + 单次最多抓多少篇
4. 保存后，调度器会按全局间隔（默认 30 分钟）跑一次
5. 也可以点「立即抓取」手动触发

> 抖音和小红书的关键词是分开配置的，互不污染。

### 4.3 博主追新

订阅你想跟踪的博主，对方发新作品时第一时间通知你。

**怎么用**：
1. 进入「小红书 → 博主追新」
2. 点「+ 订阅博主」
3. 粘贴博主主页 URL（小红书：`xiaohongshu.com/user/profile/...`；抖音：`douyin.com/user/...`）
4. 保存后扩展会立即抓一次，作为"基线"
5. 之后每 1 小时（可在「追新设置」里调）检查一次新作品
6. 检测到新作品立即发飞书消息（如果开了推送）

**卡片信息**：粉丝数、获赞、作品数、最新作品标题。

**抓取频率提醒**：抖音的博主主页 API 走懒加载，单次抓取需 30-40 秒，调度器自动控制频率，不要手动连点。

## 5. 告警接入（飞书 / 企业微信）

告警有两种发送途径，你可以同时用：

### 5.1 个人账号级别（适合一人用）

1. 进入「个人设置」
2. 找到「飞书 / 企业微信 Webhook」
3. 粘贴你自己创建的群机器人 Webhook URL
4. 点「测试推送」验证连通

适合所有告警都进同一个群的场景。

### 5.2 监控分组级别（适合多业务线）

如果你有多个项目，希望「美妆账号告警」和「数码账号告警」分开发到不同群，用监控分组：

1. 进入「小红书 → 监控帖子」（或抖音 — 分组按平台隔离）
2. 顶部点「监控分组」按钮
3. 新建分组：填名字（例如「美妆推广」）+ 该分组专属的飞书 Webhook 或 chat_id
4. 保存
5. 添加帖子时把新帖子归到这个分组
6. 该分组下的告警就只发到你配置的群，不影响其他分组

> 飞书内部群应用机器人（`feishu_chat_id`）优先级高于自定义机器人 webhook。如果你公司用的是飞书企业版，推荐用 chat_id。

### 5.3 告警阈值

「监控分组」对话框内可设置每个分组的告警阈值：
- 点赞增量阈值（默认 50）
- 收藏增量阈值（默认 50）
- 评论增量阈值（默认 1）
- 累计达标提醒（例如：点赞首次达到 1万 / 10万 / 100万）
- 24h 涨幅 ≥ N% 提醒

每条告警 4 小时内不重复推送（避免短期波动反复打扰）。

## 6. 工具区

左侧菜单「工具」下面：

### 6.1 商品图（自创）
上传一张商品图，AI 重新绘制成多种风格（小红书风、抖音封面风等）。
适合做小红书 / 抖音的封面快速生成。

### 6.2 作品仿写
粘贴一段爆款文案，选一个平台模板（小红书 / 抖音），AI 改写成对应平台的语言风格。
也可以保存自己的 Prompt 模板。

## 7. 数据导入

把已有的帖子链接 / 博主列表批量导入到平台：
1. 进入「数据导入」
2. 选导入类型（帖子 URL / 博主 URL / 关键词列表）
3. 粘贴 CSV 或纯文本（每行一条）
4. 选目标平台 + 分组
5. 点导入

## 8. 常见问题（FAQ）

**Q1. 扩展显示「未连接」怎么办？**
1. 检查工具栏图标里的服务器地址 + Token 是否正确
2. 确认服务器在线（Dashboard 能打开就在线）
3. 点「重连」，或重启 Chrome
4. 实在不行，到 `chrome://extensions/` 把扩展「重新加载」

**Q2. 抓取失败「未捕获到目标接口响应」？**
- 这是平台懒加载导致的常见情况，重试 1-2 次大多就好
- 如果反复失败：浏览器里手动打开那条链接看是否被风控（要过滑块）
- 抖音博主主页特殊：单次抓取约 30-40 秒，请耐心

**Q3. 抓取失败「需要登录」？**
浏览器里那个平台没登录或登录态过期了。打开对应站点重新登录一下即可。

**Q4. 抓取失败「触发滑块验证」？**
抓取频率太高被风控。在 Dashboard 「监控节奏」里把检查间隔调长（默认 30 分钟，可以调到 60 分钟），或者降低订阅博主数量。

**Q5. 飞书推送没收到？**
1. 在「个人设置」点「测试推送」看是否能收到
2. 检查 Webhook URL 是否过期（飞书机器人有有效期）
3. 如果设了「监控分组」级别 webhook，注意不会再发到「个人设置」的 webhook（per-group 优先）

**Q6. 我能监控多少条帖子？多少个博主？**
- 默认配额：帖子 ≤ 100 条，博主 ≤ 20 个
- 配额可在「个人设置」查看
- 超出请联系我们升级套餐

**Q7. 关闭浏览器后还会抓数据吗？**
不会。所有抓取都通过你浏览器里的扩展进行，浏览器关了 / 扩展未连接，调度器会跳过该用户的任务（不会报错，只是当次不抓）。建议设个开机启动 Chrome + 钉住扩展。

**Q8. 我的数据安全吗？**
- 所有抓取在你本地浏览器执行，平台只接收抓到的公开数据（点赞数、评论等）
- 你的登录 cookie 不会上传到服务器
- 数据库里只存帖子互动数据 + 你的飞书 webhook（webhook 已加密存储）
