# 需求文档: 小红书帖子数据监控工具

## 1. 概述

**一句话描述**：一个 Docker 化的 Web 应用，批量监控小红书帖子的点赞、收藏、评论数据，异常时自动推送企业微信通知。

**核心价值**：解放内容创作者每天手动查看多个帖子数据的重复劳动，第一时间感知爆款信号或异常互动。

**目标用户**：小红书内容创作者，每天发布多篇帖子，需要高效管理帖子表现。

---

## 2. 需求与用户故事

### 2.1 帖子管理

- **US-01**：作为用户，我可以在 Web 界面批量粘贴多个小红书帖子链接（每行一个），一键添加到监控列表。
- **US-02**：作为用户，我可以查看所有被监控帖子的列表，包含标题、当前点赞/收藏/评论数、最后更新时间。
- **US-03**：作为用户，我可以删除不再需要监控的帖子。
- **US-04**：作为用户，我可以查看单个帖子的历史数据折线图（点赞、收藏、评论随时间变化）。

**验收标准**：
- 支持同时添加 ≥ 50 个帖子链接
- 列表按最新更新时间排序
- 帖子标题自动从链接解析填充

### 2.2 数据采集

- **US-05**：系统每 30 分钟自动抓取所有监控帖子的最新数据。
- **US-06**：用户在 Web 界面配置自己的小红书账号 Cookie，系统用此 Cookie 模拟已登录状态进行抓取。
- **US-07**：当 Cookie 失效时，Web 界面展示明显警告，并推送通知提醒用户更新。

**验收标准**：
- Cookie 通过 Web 界面配置，不写死在代码中
- 抓取失败时记录错误日志，不影响其他帖子
- 支持手动触发立即采集

### 2.3 异常检测与推送

- **US-08**：当单次采集周期内某帖子**点赞或收藏增量超过阈值**（默认 50，可配置）时，推送企业微信通知。
- **US-09**：当某帖子**出现任何新评论**时，推送企业微信通知，包含评论内容摘要。
- **US-10**：每天早上 **09:00** 发送日报，包含所有帖子前一天的数据汇总（总增量排行）。

**推送内容格式示例**：
```
📈 爆款预警
帖子：《xxx标题》
点赞 +120 | 收藏 +87
查看帖子：https://...
```

**验收标准**：
- 企业微信 Webhook URL 在 Web 界面配置
- 异常阈值可在 Web 界面修改（默认 50）
- 同一帖子在同一采集周期内只推送一次
- 日报固定 09:00 发送，时区 UTC+8

### 2.4 系统配置

- **US-11**：用户在 Web 界面的「设置」页配置：Cookie、企业微信 Webhook URL、异常增长阈值。
- **US-12**：配置保存到持久化存储（数据库或配置文件），重启 Docker 后不丢失。

## 4. 技术约束

| 项目 | 选型 |
|------|------|
| 运行环境 | Docker Compose |
| 后端 | Python (FastAPI) |
| 前端 | Vue 3 + Vite（或轻量 Jinja2 模板） |
| 数据库 | SQLite（持久化 volume） |
| 调度器 | APScheduler（内嵌于 FastAPI 服务） |
| 数据抓取 | httpx + Cookie 注入（优先），失败降级 Playwright |
| 推送 | 企业微信机器人 Webhook（HTTP POST） |
| 端口 | 默认 8080，可通过环境变量配置 |
